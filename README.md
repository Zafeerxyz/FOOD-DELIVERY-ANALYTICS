# FOOD-DELIVERY-ANALYTICS
Database Schema The database follows a normalized relational schema to minimize data redundancy and maintain data integrity. Relationships One customer can place multiple orders. One restaurant can receive multiple orders. One order can contain multiple menu items. Each menu item belongs to a single restaurant. Each order is assigned to one delivery partner. Primary and foreign key relationships were implemented to enforce referential integrity across all tables.

Dataset & Technical Overview Dataset Summary The database was designed to simulate the operations of a real-world food delivery platform. It contains transactional and master data required to analyze customer behavior, restaurant performance, sales trends, delivery efficiency, and menu popularity.
