# Food Delivery Analytics using PostgreSQL

An end-to-end SQL analytics project that demonstrates how PostgreSQL can be used to transform raw transactional data into meaningful business insights for a food delivery platform.



##  Project Overview

This project simulates a real-world food delivery business by designing a normalized relational database and performing business analytics using PostgreSQL.

The primary objective is to answer key business questions related to:

- Sales Performance
- Customer Behavior
- Restaurant Performance
- Delivery Efficiency
- Revenue Trends

The project demonstrates both SQL fundamentals and advanced analytical SQL techniques commonly used in real-world business environments.



##  Project Objectives

- Design a normalized relational database.
- Analyze transactional data using SQL.
- Answer real-world business questions.
- Implement reusable SQL Views.
- Optimize reporting using Materialized Views.
- Apply advanced SQL techniques including Window Functions.
- Generate business insights and recommendations.



##  Technologies Used

- PostgreSQL
- SQL
- pgAdmin 4



##  Database Schema

The database follows a **Normalized Relational Schema (3NF)** and consists of six interconnected tables.

| Table | Description |
|---------|-------------|
| Customers | Customer information |
| Restaurants | Restaurant details |
| Menu Items | Food items offered by restaurants |
| Orders | Customer order transactions |
| Order Items | Individual items within each order |
| Delivery Partners | Delivery partner information |



##  Dataset Summary

| Table | Records |
|---------|---------:|
| Customers | 30 |
| Restaurants | 15 |
| Menu Items | 75 |
| Orders | 100 |
| Order Items | ~250 |
| Delivery Partners | 10 |

#  Views

### monthly_sales_view

Provides monthly revenue, total orders, and average order value for reporting purposes.

### restaurant_performance_view

Summarizes restaurant performance by calculating total orders, revenue, and average delivery time.



#  Materialized View

### customer_spending_mv

Stores precomputed customer spending metrics and customer segments, reducing computation time for analytical reports.
